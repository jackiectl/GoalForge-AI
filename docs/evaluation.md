# GoalForge — 训练 / 验证 / 测试与评估方法论 (Training, Validation, Test & Evaluation)

> 回答「train/val/test 怎么切、怎么判断模型好不好、怎么保证泛化」。核心：足球里球员/教练/球队
> 千变万化，**泛化主要由"表示"决定，其次由评估协议保证**；而评估协议的第一原则是**按时间切，绝不随机切**。

## 1. 铁律：按时间切 (temporal split)，绝不随机切
随机划分会把**未来**的球队状态/结果泄漏进"过去"的训练集，让模型"记住"测试期的强弱 → 分数虚高、上线崩。
所有划分、所有滚动特征、时间衰减的参考日，**只能用比赛日之前的信息**。

## 2. 数据划分方式

**(a) 时间留出 (temporal holdout) —— 主干**
按日期排序全部比赛：
- **Train** ≈ 最早 70%：拟合模型参数。
- **Validation** ≈ 之后 15%：调超参（收缩 `l2`、时间衰减 `xi`、球员先验强度、`n_sims` 等）。可反复看。
- **Test** ≈ 最新 15%：**锁起来，最后只评一次**，不参与任何决策。

**(b) Walk-forward 扩窗回测 —— 真正的评估**
对 val/test 期的每个区块：用**严格早于它**的所有数据重拟合 → 预测 → 打分。给出大量样本外预测，数据利用率高，
是体育/时序预测的金标准（本项目 `evaluation/backtest.py` 就是这个）。对"会重训"的统计模型，这**就是** test。

**(c) 泛化压力测试（因为球员/教练/球队会变）**
- **留一赛事 / 留一赛季 (leave-one-tournament-out)**：整届 WC2022 完全不进训练，去预测它 → 衡量对**没见过的赛事**的域外泛化。
- **冷启动切片 (cold-start slices)**：按"参赛队/球员的历史样本量"给 test 分层，分别报 RPS；若在"少见实体"上明显更差，说明泛化弱。
- **换人鲁棒性**：把低历史球员换进首发（先验兜底），预测应仍合理、不炸。

**(d) 防泄漏清单**
- 特征只用赛前信息（滚动统计截到赛前一天）。
- 时间衰减参考日 = 该场比赛日（不给未来加权）。
- 算某场的球员 per-90 率时，只用该球员**此前**的出场；walk-forward 里对赛前数据重算。

## 3. 泛化从哪来：表示决定泛化 (representation ⇒ generalization)
- **纯球队参数（DC 的 attack/defence）不泛化到新队**，且无法解释"这队强是因为球员强"。它是"域内、需重估"的。
- **要泛化到"任意首发"**：用**球员的可迁移特征**（per-90 得分/助攻率、VAEP/xT、身价）+ **教练特征**聚合成球队强度，
  这样没见过的阵容也能算分；**新球员冷启动**用位置/全局先验 + 收缩兜底。
- **跨域（国家队 → 俱乐部）**：球队级参数按域**分别重训**；但**球员级表示可迁移**——这正是先做球员层的意义。

## 4. 怎么判断模型好不好

**指标（概率预测，校准和区分度同等重要）**
- 1X2：**RPS（主）**、log-loss、Brier、**校准 ECE / 可靠性曲线**。
- 球员层：**top-k 射手/助攻命中率**、**anytime 概率校准**（预测 30% 的球员，是否约 30% 真进球）。

**必须打过的基线（诚实性关键）**
1. **base-rate**（主/平/客历史频率）——地板。
2. **Elo / 身价**——强简单基线。
3. **博彩去水位赔率**——天花板；能逼近就很好，"大幅稳定跑赢庄家"基本不现实，出现即怀疑泄漏。

**Ablation & 稳定性**
- 加**阵容/球员层**到底比"只用球队"强吗？（*Betting the system* 2022 警告：常常没强多少 → 必须实测，别假设。）教练信息有用吗？
- 多赛季/多赛事重复，报 **均值 ± 波动**，别只看单次划分。

## 5. Checkpoint / 推理 / fine-tune 流程
- **训练一次** → 把拟合好的参数序列化到 `models/<name>.pkl`（统计模型就是一堆 float 字典，很小）。
- **推理** = load checkpoint + 跑蒙特卡洛，秒级、不重拟合。
- **更新** = 数据变多时 refit / warm-start（统计模型很便宜）；未来神经模型才是真正的 checkpoint **fine-tune**。
- checkpoint 以**数据截止日**命名 → 预测可复现。

## 6. 本项目的落地方案
| 层 | 训练数据 | 划分 / 评估 | 泛化点 |
|---|---|---|---|
| **团队比分层** | **martj42 国际赛结果**（CC0，~4.9 万场，覆盖所有国家队、跨几十年） | 时间留出 + walk-forward；留一届 WC | 跨所有国家队 + 时间衰减跟强弱漂移 |
| **球员进球/助攻层** | **StatsBomb 事件**（WC/Euro/联赛，含分钟） | per-90 率 + 经验贝叶斯收缩 | 覆盖大量球员；冷启动用先验 |
| **域切换（俱乐部）** | Understat/FBref/Transfermarkt | 同上，团队参数按域重训 | 球员表示迁移 |

> 说明：martj42 无阵容/分钟，故只训**团队层 + 射手**；**球员层用 StatsBomb**。二者产出各自的 checkpoint，
> 推理时一起 load。Bayesian 全后验（NumPyro/GPU）作为团队层的升级项，在**独立环境 + Slurm GPU 节点**上训。

## 7. Phase 3 实验：阵容感知神经模型（诚实的负结果）
**问题**：知道具体首发 11 人,能不能预测得比只知道球队更好? **做法**:一个用梯度下降拟合的广义
Dixon-Coles(PyTorch/GPU,`scripts/train_neural.py`)——球队的 attack/defence = 球队级评分 **+ 首发 XI
每名球员 attack/defence 增量的均值**(带位置基线),球员增量强 L2 收缩,训练集没见过的球员保持 0 增量、
回退到球队+位置。数据:6 届现代国际大赛的 **314 场带首发 XI**(StatsBomb,`scripts/build_intl_lineups.py`)。
超参**先验固定**(不在测试折上调)。评估:**留一届赛事**(train 5 届、预测第 6 届),对比团队级 DC / Elo / 基准率。

**结果(留一赛事,加权总体,越低越好):**

| 模型(同数据训练) | RPS | logloss | ECE(校准) |
|---|---|---|---|
| **Elo** | **0.225** | **1.08** | 0.093 |
| Base-rate | 0.231 | 1.09 | **0.044** |
| Dixon-Coles | 0.245 | 1.20 | 0.166 |
| Neural(阵容感知) | 0.246 | 1.20 | 0.207 |

**诚实解读**:阵容感知神经模型**没有**打赢团队级 Dixon-Coles(基本打平),而且**两者都输给简单的 Elo,甚至
输给基准率**;复杂模型还过度自信(ECE 更高)。世界杯单场结果高方差、可用样本少(参赛队实力接近、爆冷多),
额外的模型复杂度只带来过拟合。这与 #3a 的发现一致(DC 在留一届 WC 上过度自信)。

**注意口径**:此处为公平起见,Elo/DC/Neural 都**只用这 314 场赛事数据**训练;而**线上部署的 DC 用的是
martj42~1.6 万场**(数据多得多),在更宽的留出集上 RPS 0.173、胜过 Elo——但在纯赛事留一里同样过度自信。
阵容感知模型受限于「带阵容的比赛只有 ~314 场」(martj42 无阵容),这是它的数据天花板。

**跟进实验(把上面「留待后续」的两条都做了):**

**(1) 强团队先验 + 阵容增量**(`scripts/train_neural_v2.py`)。团队基线换成 martj42 **1.6 万场**拟合的 DC
(留一届时按日期+参赛队**精确剔除该届**、防泄漏),在这个固定强基线上再学球员增量。留一赛事加权:

| 模型 | RPS | logloss | ECE |
|---|---|---|---|
| **DC-prior(强先验)** | **0.2004** | **0.999** | 0.132 |
| DC + 阵容增量 | 0.2013 | 1.006 | 0.138 |
| Elo | 0.2029 | 1.014 | 0.135 |
| Base | 0.2334 | 1.099 | 0.077 |

两点:① **强先验 DC 反超了 Elo**(0.2004 < 0.2029)——第一版「DC 输 Elo」纯粹是**训练数据太少**的假象,不是模型问题;
② **加阵容增量反而略差**(0.2013 ≥ 0.2004),即有了好的球队模型后,具体首发对**比分**不再增加信号。

**(2) 俱乐部数据扩样本**(`scripts/build_club_lineups.py` + `scripts/train_neural_v3.py`)。再并入 **700 场
俱乐部比赛**(La Liga/英超等,球员与国家队高度重叠),让球员增量见到 **3 倍**首发样本;国际赛用 DC 先验、俱乐部赛
用可学习球队嵌入、球员增量**共享**。留一赛事加权:**DC+阵容(含俱乐部)RPS 0.2028 vs DC-prior 0.2004**——**3 倍
数据仍不翻盘**,增量依旧不加分。

**最终诚实结论**:三段实验一致——在「**比分**」层面,知道具体首发 11 人**不比一个好的球队强度模型更强**(样本再多也一样)。
**决策:线上保持团队层 DC(martj42)+ 球员分配层**,阵容神经增量**不投产**。GPU 神经路线本身跑通了(Slurm/CUDA 正常),
但诚实地说当前它不带来增益。注意:首发**确实**影响「**谁**进球/助攻」——那由线上的**射手/助攻分配层**(用 XI)负责、已在产品里;
阵容对「**多少**球(比分)」的边际贡献,在国际大赛这种高方差小样本域里被球队强度盖过了。
